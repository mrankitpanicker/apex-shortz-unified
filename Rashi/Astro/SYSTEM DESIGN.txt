Astro — Headless AI Video Production Pipeline
Portfolio Case Study
1. Executive Summary (Non-Technical)
Astro is a fully automated, offline system that generates broadcast-ready daily horoscope (Rashifal) videos — complete with AI-cloned Hindi narration, word-synced karaoke subtitles, and professional video rendering — and publishes them to YouTube without any human interaction.

Why it matters: Producing a daily video that covers all 12 zodiac signs — with scripted narration, timed subtitles, and polished rendering — typically requires hours of voice recording, manual subtitle timing, editing, and uploading. Astro eliminates the entire workflow. It runs on a schedule, generates the full video autonomously, and publishes it to a live YouTube channel.

Real-world value:

Replaces a multi-hour daily production workflow with a zero-touch automated pipeline
Generates unique daily content across 12 zodiac signs using structured randomization, ensuring no two days produce identical scripts
Runs entirely on local hardware — zero cloud costs, zero API rate limits, zero data exposure
Operates as a scheduled unattended system via Windows Task Scheduler — no manual trigger required
Currently in active daily production, generating content for a live YouTube channel

Key Technical Decisions
XTTS v2 for on-device voice cloning: Chose Coqui XTTS over cloud TTS APIs (ElevenLabs, Google Cloud TTS) to maintain a consistent voice identity across every video with zero per-request cost. At daily production volume, cloud TTS would add significant recurring cost with no quality advantage for this use case.

Whisper for word-level timestamp extraction: OpenAI Whisper provides word-level timestamps as part of transcription, eliminating the need for a separate forced-alignment tool. This gives dual value: quality verification (the system can detect if TTS produced garbled output) and precise subtitle timing data.

ASS subtitle format with karaoke tags: Chose Advanced SubStation Alpha over SRT/WebVTT because ASS supports per-word \kf karaoke highlighting with fade effects, border/shadow/blur control, and precise font sizing — producing a professional look that SRT cannot achieve.

Structured randomization over LLM generation: The script generator uses categorized phrase pools with logical connectors rather than an LLM. This guarantees grammatically correct output, eliminates hallucination risk, ensures consistent tone, and requires zero inference cost for text generation.

Lossless concat demuxer for final merge: Individual rashi segments are merged using FFmpeg's concat demuxer with -c copy, avoiding re-encoding 12 segments and saving significant processing time while maintaining quality.

History journaling for content diversity: A JSON-based history system tracks which phrases were used on which dates, with a configurable retention window (10 days). This prevents the system from repeating the same horoscope text across consecutive days.

3. Engineering Challenges Solved
Multi-Model AI Pipeline Orchestration
Sequential dual-model inference: Each of the 12 rashi segments requires loading and running XTTS v2 (speech synthesis) followed by Whisper (speech recognition for alignment). The pipeline manages GPU memory across both models, with automatic CUDA detection and CPU fallback.
Chunk-boundary management for TTS stability: XTTS v2 produces unstable output beyond ~200 characters. The word-safe text splitter (
split_text_safe
) ensures text is never cut mid-word or mid-sentence, and the concatenation pipeline merges chunk WAVs with volume normalization (dynaudnorm) and gain boosting.
Subtitle Timing Precision
Word-count mismatch handling: XTTS synthesizes audio from Hindi text, but Whisper's transcription often produces a different word count than the original script. The alignment engine maps Whisper timestamps to the reference text using min(ref_words, timestamps), ensuring subtitles always display the authored script — not Whisper's imperfect transcription.
Multi-layered timing offsets: The subtitle system manages four distinct timing parameters — a global sync offset (1.5s), a subtitle start delay (4.0s matching the audio delay), inter-chunk hold time (0.5s), and inter-chunk spacing (0.5s) — all coordinated to keep visual highlights synchronized with spoken audio.
Karaoke-style word highlighting: Each subtitle word carries its own \kf duration in centiseconds derived from Whisper timestamps, with a per-word inter-delay (3cs) to prevent the highlight from racing ahead. Dynamic fade-in/fade-out durations scale proportionally to chunk length.
Audio Engineering
Volume normalization pipeline: Raw XTTS output varies in volume across chunks. FFmpeg's dynaudnorm filter with a 75-frame window normalizes loudness, and a 4x gain multiplier ensures the final audio is broadcast-ready without clipping.
Audio-video sync with delayed injection: The rashi audio is injected with a configurable delay (4 seconds) using FFmpeg's adelay filter, creating a silent intro period that matches the video's visual opening before narration begins.
Content Diversity Engine
Six-category structured generation: Each rashi script is assembled from 6 independent phrase pools (positive status, negative warning, remedies, general status, auspicious colors, luck metrics) with logical connectors ("वहीं," / "हालांकि,") for natural paragraph flow.
Hindi-specific text processing: Numbers are converted to Hindi words for better TTS pronunciation (e.g., 60% → साठ प्रतिशथ). The Devanagari punctuation mark । is padded for proper TTS pausing, and common mispronunciation patterns are corrected before synthesis.
YouTube Publishing Automation
OAuth2 with automatic token refresh: The upload module handles the full OAuth2 lifecycle — loading cached tokens, refreshing expired credentials, and falling back to browser-based re-authorization when needed. Credentials persist across runs without re-authentication.
Resumable upload with progress tracking: Large video files (130+ MB) are uploaded using Google's resumable upload protocol with real-time upload percentage logging, ensuring uploads survive transient network interruptions.
Auto-generated metadata: Video title, description, and tags are dynamically populated with the current date in Hindi format, SEO-optimized keywords, and proper YouTube category classification.

Designed an audio normalization pipeline that produces consistent broadcast-ready volume levels from variable TTS output

Decisions I Made
Structured randomization over LLM text generation: Guaranteed grammatically correct, tonally consistent output with zero inference cost — LLMs would add latency, hallucination risk, and compute overhead for a domain where templated generation produces better results
XTTS v2 over cloud TTS: Eliminated recurring costs and maintained voice identity consistency without depending on third-party API availability
ASS over SRT: Chose the subtitle format that supports the karaoke highlighting effect that makes the videos feel professional and polished
History journaling over database: A simple JSON file tracks used phrases with date-based expiration — the right amount of persistence for a single-machine daily workflow, without the overhead of a database server
Trade-offs I Handled
Quality vs. throughput: XTTS v2 produces high-quality cloned speech but is slower than lighter TTS models. The 200-character chunk limit was tuned to balance synthesis stability against processing time — smaller chunks are more reliable but produce more FFmpeg merge operations.
Whisper accuracy vs. simplicity: Whisper occasionally miscounts words in Hindi text, requiring the alignment engine to handle word-count mismatches. This trade-off was accepted because Whisper eliminates the need for a separate forced-alignment tool and provides verification capability as a bonus.
Deterministic text vs. generative variety: Structured randomization from phrase pools produces less variety than LLM generation, but guarantees tone consistency, grammatical correctness, and predictable runtime — critical properties for an unattended daily system.

6. Impact Framing
System Design Skills
Astro demonstrates the ability to decompose a complex media production workflow into a clean multi-stage pipeline where each component (text generation, voice synthesis, audio processing, subtitle alignment, video rendering, publishing) has well-defined inputs, outputs, and failure boundaries. Each rashi segment is processed independently — a failure in one segment doesn't block the remaining eleven.

Production Thinking
This is not a prototype or a demo. It is a system that runs daily on a schedule, generates real content, and publishes to a live YouTube channel. The history journaling, automatic token refresh, resumable uploads, per-rashi error resilience, and temp file cleanup all reflect production engineering — designing for the day the system fails, not just the day it works.

Cost & Efficiency Engineering
The local-first architecture eliminates recurring cloud API costs entirely. At daily production volume, cloud TTS pricing (ElevenLabs, Google Cloud TTS) would scale linearly with output. With Astro, once the hardware investment is made, the marginal cost of each additional video is effectively zero — electricity and internet bandwidth only.

AI Engineering Depth
Running two distinct AI models (speech synthesis + speech recognition) in a coordinated pipeline, managing GPU resources across both, and building a robust word-alignment layer that reconciles their output differences demonstrates applied ML engineering. This is not API consumption — it is on-device model orchestration with domain-specific adaptation (Hindi pronunciation correction, number-to-word conversion, sentence-safe chunking).

Scalability Mindset
The architecture is inherently parallelizable — each rashi segment is independent and could run concurrently with minimal refactoring. The config-driven design (rashi list, video mapping, timing parameters) means adding new content categories or changing output format requires configuration changes, not code rewrites.

Senior-Level Engineering Judgement
Every technical decision has a clear rationale grounded in the system's operational constraints: structured randomization over LLMs (correctness > variety for unattended systems), XTTS over cloud TTS (cost elimination + voice consistency), ASS over SRT (karaoke capability), JSON history over database (right-sized persistence), lossless concat over re-encode (speed over flexibility). This is the hallmark of senior engineering — choosing the right tool for the constraint set, not the most impressive one.
